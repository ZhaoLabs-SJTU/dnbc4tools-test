## 第一章：你需要知道的基础概念

### 1.1 华大单细胞上游分析是什么？

华大（DNBSEQ）单细胞转录组测序会产生 FASTQ 文件（测序原始数据），上游分析就是把这些 FASTQ 文件处理成基因表达矩阵（每个基因在每个细胞里的表达量）。

**类比**：FASTQ 是原始食材 → 上游分析是烹饪 → 基因表达矩阵是成品菜

### 1.2 四个核心步骤

```
FASTQ 文件
    │
    ▼
Step 1: rna data     质控 + 比对 + 基因注释  (~1-8 小时)
    │                    产出: BAM 文件
    ▼
Step 2: rna count    计数 + 细胞鉴定        (~0.5-1 小时)
    │                    产出: 基因表达矩阵
    ▼
Step 3: rna analysis 降维 + 聚类 + 细胞注释  (~0.5 小时)
    │                    产出: h5ad 文件
    ▼
Step 4: rna report   生成 HTML 报告         (~5 分钟)
                         产出: 网页报告
```

### 1.3 关键术语速查

| 术语 | 大白话解释 |
|------|----------|
| **FASTQ** | 测序仪输出的原始数据文件 |
| **参考基因组（FASTA）** | 该物种的"标准基因序列"，好比字典 |
| **GTF 注释文件** | 标注了基因在基因组上的位置，好比字典的页码索引 |
| **STAR 索引** | 把参考基因组"预处理"成可以快速搜索的格式，好比给字典建了目录 |
| **BAM** | 比对结果文件，记录了每条测序读段比对到基因组的哪个位置 |
| **基因表达矩阵** | 行=基因，列=细胞，值=该基因在该细胞中的表达量 |
| **h5ad** | 标准单细胞数据格式（AnnData），下游分析（Seurat/Scanpy）可直接读取 |
| **conda** | Python 环境管理器，类似手机的应用商店，每个项目可以有不同的 App |

## 第二章：环境搭建（从零开始）

### 2.1 安装 Miniconda（如果已有 conda 可跳过）

⏱️ **耗时**：约 10 分钟

```bash
# 下载 Miniconda 安装脚本
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh

# 安装（按提示操作：Enter → yes → yes）
bash Miniconda3-latest-Linux-x86_64.sh

# 让 conda 生效
source ~/.bashrc
```

✅ **预期输出**：`conda --version` 显示版本号（如 `conda 24.x.x`）

❌ **常见错误**：
| 错误 | 解决方法 |
|------|---------|
| `wget: command not found` | 用 `curl -O` 替代，或 `sudo apt install wget -y` |
| `conda: command not found` | 重新登录终端，或 `source ~/miniconda3/etc/profile.d/conda.sh` |

### 2.2 创建 dnbc4tools 专用环境

⏱️ **耗时**：约 5-10 分钟

> ⚠️ **重要**：请务必创建独立的 conda 环境！不要在 base 环境里安装，否则会污染基础环境。

```bash
# 创建名为 dnbc4tools 的独立环境（Python 3.9）
conda create -n dnbc4tools python=3.9 -y

# 激活环境
conda activate dnbc4tools

# 验证：命令提示符前面应该显示 (dnbc4tools)
which python
# 预期输出: /home/你的用户名/miniconda3/envs/dnbc4tools/bin/python
```

✅ **预期输出**：提示符变为 `(dnbc4tools) 用户名@服务器:~$`

### 2.3 安装 dnbc4tools

⏱️ **耗时**：约 5 分钟

```bash
# 确认已在 dnbc4tools 环境
conda activate dnbc4tools

# 安装 dnbc4tools
pip install dnbc4tools -i https://pypi.org/simple/

# 验证安装
dnbc4tools --version
# 预期输出: dnbc4tools, version 2.1.3

# 查看 rna 子命令
dnbc4tools rna --help
```

❌ **常见错误**：
| 错误 | 解决方法 |
|------|---------|
| pip 下载慢 | 换镜像源：`pip install dnbc4tools -i https://pypi.tuna.tsinghua.edu.cn/simple` |
| 依赖冲突 | 先 `pip install --upgrade pip setuptools wheel` |
| 需要特定版本 | `pip install dnbc4tools==2.1.3` |

### 2.4 安装 mamba（可选，但强烈推荐）

mamba 是 conda 的加速版，安装依赖快得多：

```bash
conda install -n base -c conda-forge mamba -y

# 之后可以用 mamba 替代 conda
mamba create -n dnbc4tools python=3.9 -y
```

## 第三章：准备参考基因组

> ⚠️ 参考基因组只需准备一次，所有样本共用同一套。

### 3.1 下载参考基因组

不同物种的下载地址不同。以下为 Ensembl 快查表：

| 物种 | 基因组大小 | Ensembl 版本 | 下载地址 |
|------|:--:|------|------|
| 人 (Homo sapiens) | 3.2 Gb | GRCh38 | https://ftp.ensembl.org/pub/release-113/fasta/homo_sapiens/dna/ |
| 小鼠 (Mus musculus) | 2.7 Gb | GRCm39 | https://ftp.ensembl.org/pub/release-113/fasta/mus_musculus/dna/ |
| 大鼠 (Rattus norvegicus) | 2.6 Gb | mRatBN7.2 | https://ftp.ensembl.org/pub/release-113/fasta/rattus_norvegicus/dna/ |
| 食蟹猴 (Macaca fascicularis) | 2.9 Gb | Macaca_fascicularis_6.0 | https://ftp.ensembl.org/pub/release-113/fasta/macaca_fascicularis/dna/ |
| 斑马鱼 (Danio rerio) | 1.4 Gb | GRCz11 | https://ftp.ensembl.org/pub/release-113/fasta/danio_rerio/dna/ |
| 猪 (Sus scrofa) | 2.5 Gb | Sscrofa11.1 | https://ftp.ensembl.org/pub/release-113/fasta/sus_scrofa/dna/ |

#### 下载示例（以人为例）

```bash
# 创建参考基因组目录
mkdir -p Ref && cd Ref

# 下载基因组 FASTA（主文件）
wget https://ftp.ensembl.org/pub/release-113/fasta/homo_sapiens/dna/Homo_sapiens.GRCh38.dna.primary_assembly.fa.gz

# 下载 GTF 注释文件
wget https://ftp.ensembl.org/pub/release-113/gtf/homo_sapiens/Homo_sapiens.GRCh38.113.gtf.gz

# 解压
gunzip Homo_sapiens.GRCh38.dna.primary_assembly.fa.gz
gunzip Homo_sapiens.GRCh38.113.gtf.gz

# 重命名为标准名称（方便脚本识别）
mv Homo_sapiens.GRCh38.dna.primary_assembly.fa genome.fa
mv Homo_sapiens.GRCh38.113.gtf genes.gtf

cd ..
```

### 3.2 构建 STAR 比对索引

⏱️ **耗时**：食蟹猴约 1-2 小时，人类约 1.5-2 小时

> ⚠️ STAR 索引构建需要大量内存。食蟹猴/人类基因组峰值内存约 70-80 GB。

```bash
conda activate dnbc4tools

dnbc4tools rna mkref \
    --fasta Ref/genome.fa \
    --gtf Ref/genes.gtf \
    --out Ref/dnbc4tools_ref
```

✅ **预期输出**：`Ref/dnbc4tools_ref/` 目录下有 `Genome`、`SAindex`、`SA`、`chrLength.txt` 等文件

**检查索引完整性**：

```bash
ls -lh Ref/dnbc4tools_ref/
# 必须有的文件: Genome, SAindex, SA, chrLength.txt, chrName.txt, chrNameLength.txt, chrStart.txt

# Genome 文件大小应接近 FASTA 文件大小
du -sh Ref/dnbc4tools_ref/Genome
du -sh Ref/genome.fa
# 两者应接近（如 2.9G 对 2.9G）
```

### 3.3 硬件参考

| 物种 | 基因组大小 | 推荐内存 | 索引耗时 | 单样本耗时 |
|------|:--:|:--:|:--:|:--:|
| 斑马鱼 | 1.4 Gb | 32 GB | ~30 min | ~1.5 h |
| 猪 | 2.5 Gb | 64 GB | ~1 h | ~2.5 h |
| 大鼠 | 2.6 Gb | 64 GB | ~1 h | ~2.5 h |
| 小鼠 | 2.7 Gb | 64 GB | ~1 h | ~2.5 h |
| 食蟹猴 | 2.9 Gb | 80 GB | ~1.5 h | ~3 h |
| 人 | 3.2 Gb | 80 GB | ~2 h | ~3.5 h |

### 4.2 项目目录结构（要求）

```
你的项目目录/
├── 上游/
│   └── fastq/                        # ← FASTQ 文件放这里
│       ├── Sample01_R1_001.fastq.gz
│       ├── Sample01_R2_001.fastq.gz
│       ├── Sample01oligo_R1_001.fastq.gz
│       ├── Sample01oligo_R2_001.fastq.gz
│       ├── Sample02_R1_001.fastq.gz
│       └── ...
├── Ref/                              # ← 参考基因组放这里
│   ├── genome.fa
│   ├── genes.gtf
│   └── dnbc4tools_ref/              # ← STAR 索引
│       ├── Genome
│       ├── SAindex
│       └── ...
├── 下游/
│   └── 0.RAW/                        # ← 结果输出到这里
└── 华大批量上游分析dnbc4tools.sh      # ← 脚本放这里
```

### 4.3 FASTQ 文件命名规则

脚本支持以下命名格式：

| 类型 | R1 文件 | R2 文件 | Oligo R1 | Oligo R2 |
|------|---------|---------|----------|----------|
| **标准型** | `Sample1_R1_001.fastq.gz` | `Sample1_R2_001.fastq.gz` | `Sample1oligo_R1_001.fastq.gz` | `Sample1oligo_R2_001.fastq.gz` |
| **wk 型** | `Sample1wk_R1_001.fastq.gz` | `Sample1wk_R2_001.fastq.gz` | `Sample1oligo_R1_001.fastq.gz` | `Sample1oligo_R2_001.fastq.gz` |

> 脚本 v3.1 支持通配符回退，能自动识别大多数华大命名格式。如果不确定，先 dry-run 测试。

### 4.4 修改脚本参数（核心步骤！）

打开脚本，修改**脚本开头 `参数解析` 区域**的以下内容：

```bash
vim 华大批量上游分析dnbc4tools.sh
# 或
nano 华大批量上游分析dnbc4tools.sh
```

#### 必须修改的参数（★ = 每次新项目必改）

| 参数 | 默认值 | 如何修改 | 示例（切换为人肝项目） |
|------|--------|---------|----------------------|
| **★ FASTQ_DIR** | `./上游/fastq` | 你的 FASTQ 目录路径 | `/data/liver_project/fastq` |
| **★ OUT_DIR** | `./下游/0.RAW` | 输出目录路径 | `/data/liver_project/output` |
| **★ REF_DIR** | `./Ref` | 参考基因组目录 | `/data/liver_project/Ref` |
| **★ GTF** | `./Ref/genes.gtf` | GTF 注释文件路径 | `/data/liver_project/Ref/genes.gtf` |
| **★ MTGENES** | `./Ref/mtgenes.txt` | 线粒体基因列表路径 | `/data/liver_project/Ref/mtgenes.txt` |
| **★ SAMPLE_PREFIX** | `26SC-KY` | FASTQ 文件的样本名前缀 | `LiverSample-` |

#### 建议调整的参数

| 参数 | 默认值 | 建议 |
|------|--------|------|
| `THREADS` | `30` | 根据服务器 CPU 核数调整（建议 20-30） |
| `EXPECT_CELLS` | `3000` | 根据预估细胞数调整（emptydrops 方法下为软引导，不影响最终数） |
| `CALLING_METHOD` | `emptydrops` | 推荐保持 emptydrops；cellranger 模式会硬截断 |
| `CONDA_BASE` | `$HOME/miniconda3` | 确认你的 conda 安装路径 |
| `CONDA_ENV` | `dnbc4tools` | 确认你的 conda 环境名 |

#### mtgenes.txt 准备

mtgenes.txt 包含线粒体基因名，一行一个。常见内容：

```
MT-ND1
MT-ND2
MT-COX1
MT-COX2
MT-ATP8
MT-ATP6
MT-COX3
MT-ND3
MT-ND4L
MT-ND4
MT-ND5
MT-ND6
MT-CYTB
```

> 不同物种线粒体基因名前缀不同：人类=`MT-`，小鼠=`mt-`，大鼠=`Mt-`，斑马鱼=`mt-`。请核对你的 GTF 文件中线粒体基因的实际命名。

## 第六章：nohup 后台批量启动

### 6.1 什么是 nohup？为什么用它？

- `nohup` = "no hang up"：你关掉终端窗口后，程序继续在后台运行
- `&` = 后台运行：终端不卡住，可以继续做其他事
- `> batch_analysis.log 2>&1` = 把输出（含错误）记录到日志文件

### 6.2 标准 7 步启动

```bash
# 1. 进入项目目录
cd /your/project/path/

# 2. 激活 conda 环境
conda activate dnbc4tools

# 3. Dry-run 确认（可选但强烈推荐）
bash 华大批量上游分析dnbc4tools.sh -n

# 4. 后台启动
nohup bash 华大批量上游分析dnbc4tools.sh > batch_analysis.log 2>&1 &

# 5. 记录进程 PID
echo $! > batch.pid

# 6. 验证进程是否正常运行
ps -p $(cat batch.pid)

# 7. 查看实时日志
tail -f batch_analysis.log
```

✅ **预期输出**（`ps` 命令）：
```
  PID TTY      STAT   TIME COMMAND
12345 pts/0    S      0:00 bash 华大批量上游分析dnbc4tools.sh
```

### 6.3 常用启动变体

```bash
# === 并行 4 个样本（12 样本从 38h 缩短至 ~10h）===
nohup bash 华大批量上游分析dnbc4tools.sh -j 4 > batch_analysis.log 2>&1 &

# === 自定义路径 ===
nohup bash 华大批量上游分析dnbc4tools.sh \
    -f /data/myproject/fastq \
    -o /data/myproject/output \
    -r /data/myproject/Ref > batch_analysis.log 2>&1 &

# === 从第 5 个样本继续（前 4 个已完成）===
nohup bash 华大批量上游分析dnbc4tools.sh \
    --resume-from Sample05 > batch_analysis.log 2>&1 &

# === 强制重跑所有（忽略完成标记）===
nohup bash 华大批量上游分析dnbc4tools.sh \
    --force > batch_analysis.log 2>&1 &

# === 调试模式（打印每条命令）===
nohup bash 华大批量上游分析dnbc4tools.sh \
    --debug > batch_analysis.log 2>&1 &
```

### 6.4 命令拆解

| 命令部分 | 含义 |
|----------|------|
| `nohup` | 忽略挂断信号，关终端不中断 |
| `bash 华大批量上游分析dnbc4tools.sh` | 执行脚本 |
| `-j 4` | 最多同时运行 4 个样本 |
| `> batch_analysis.log` | 标准输出写入此文件 |
| `2>&1` | 错误输出也写入同一文件 |
| `&` | 放到后台运行 |

启动后 30 秒内执行以下检查：

```bash
# ☐ 3. 无错误信息
```

### Q3: Dry-run 显示 `cDNA: ✗`

说明 FASTQ 文件命名与脚本期望不匹配。

**解决**：

```bash
# 查看你的 FASTQ 实际命名

# 确认 SAMPLE_PREFIX 是否正确
# 例如文件 SampleAA_R1_001.fastq.gz → SAMPLE_PREFIX="Sample"

# 脚本 v3.1 支持通配符回退，会自动尝试 {样本}*_R1*.fastq.gz
# 如果仍失败，修改脚本 find_cdna_r1 函数或手动重命名 FASTQ 文件
```

### Q4: 内存不足（OOM killed）

```bash
# 症状：日志突然中断，dmesg 显示 "Out of memory"
dmesg | grep -i "killed process"

# 解决：
# - 降低并行数: -j 1（串行）
# - 减少线程数: -t 10
# - 确保服务器内存 ≥ 参考基因组大小对应推荐值
```

### Q5: 磁盘空间不足

```bash
# 每个样本约需 60 GB

```

### Q6: 某样本失败后如何重跑

```bash

# .step_4_04.report.done  → rna report
```

```bash
# 1. 准备参考基因组（如果还没有）

# 3. 准备线粒体基因列表
printf "MT-ND1\nMT-ND2\nMT-COX1\nMT-COX2\nMT-ATP8\nMT-ATP6\nMT-COX3\nMT-ND3\nMT-ND4L\nMT-ND4\nMT-ND5\nMT-ND6\nMT-CYTB\n" > mtgenes.txt

```

```
你遇到了问题 ↓
    │
    ├─ 脚本跑不起来？

    │   └─ oligo: ✗ → 检查 oligo 文件是否存在
    │
    ├─ 运行中报错？

    │
    └─ 结果异常？
        ├─ 细胞数太少 (<500) → 增加 EXPECT_CELLS 或检查数据质量
        ├─ 比对率低 (<50%) → 检查参考基因组是否正确物种
        └─ 报告空白 → 检查 Step 4 日志
```
